# 个人介绍网站 — 页面设计文档

---

## 一、设计概述

### 1.1 设计定位

| 维度 | 描述 |
|------|------|
| **风格** | 现代科技简约风 |
| **色调** | 深色背景 + 紫色渐变主色 + 青色点缀 |
| **布局** | 单页纵向滚动，每个模块占满视口 |
| **动效** | 滚动渐入、打字机、悬浮微交互 |
| **适配** | 仅桌面端（1920px / 1440px 最佳） |

### 1.2 设计关键词

> **干净 · 科技感 · 层次分明 · 微动效 · 不花哨**

| 属性 | 值 |
|------|-----|
| 高度 | `100vh` |
| 背景 | 深色渐变 + Canvas 粒子动画（可选：Three.js 3D粒子） |
| 头像尺寸 | 180×180px，圆形，`border: 3px solid` 带 `box-shadow` 紫色光晕 |
| 标题动画 | 打字机效果：逐字显示 + 闪烁光标，循环播放 |
| CTA 按钮 | 主按钮（渐变紫）+ 次按钮（边框透明），hover 发光 |
| 滚动指示 | 底部居中，上下弹跳动画 |

**打字机动画实现思路：**
```
CSS 方案: @keyframes typing { from { width: 0 } to { width: 100% } }
          配合 steps() 步进函数 + border-right 光标闪烁
JS 方案:  setInterval 逐字追加 + "|" 光标 blink
```

## 六、动效清单

| 序号 | 动效名称 | 触发条件 | 实现方式 | 优先级 |
|------|----------|----------|----------|--------|
| 1 | 滚动渐入 | 元素进入视口 | Intersection Observer + CSS `opacity`/`translateY` 过渡 | ⭐ 必须 |
| 2 | 打字机 | 页面加载完毕 | CSS `steps()` 动画 / JS `setInterval` | ⭐ 必须 |
| 3 | 导航高亮 | 滚动切换模块 | Intersection Observer 监听各 section | ⭐ 必须 |
| 4 | 卡片悬浮 | 鼠标悬停 | CSS `transform: translateY(-4px)` + `box-shadow` | ⭐ 必须 |
| 5 | 粒子背景 | 持续运行 | Canvas 2D / Three.js Points | ⭐⭐ 推荐 |
| 6 | 进度条 | 页面滚动 | `window.scrollY` → 百分比 | ⭐⭐ 推荐 |
| 7 | 数字递增 | 进入视口 | JS `requestAnimationFrame` 数值动画 | ⭐⭐ 推荐 |
| 8 | 头像光晕 | 持续 | CSS `@keyframes` 旋转渐变光圈 | ⭐⭐ 推荐 |
| 9 | 主题切换 | 按钮点击 | CSS 变量切换（暗黑↔亮色） | ⭐⭐⭐ 加分 |

## 八、技术实现方案

### 8.1 推荐技术栈

```
框架:     React 18 + Vite
样式:     CSS Modules + CSS 变量（零依赖）
动效:     纯 CSS animation + Intersection Observer API
粒子:     Canvas 2D（可选 Three.js）
字体:     Google Fonts / 系统字体栈
部署:     Vercel（git push 自动部署）
```

### 8.3 组件通信（零依赖）

```
数据流向: 单向，自上而下

profile.js (数据中心)
    │
    ├──> App.jsx (拉取数据，分发给各组件)
    │       │
    │       ├──> Hero        ({ name, title, subtitle })
    │       ├──> About       ({ bio, stats })
    │       ├──> Skills      ({ skills[] })
    │       ├──> Projects    ({ projects[] })
    │       ├──> Contact     ({ email, social[] })
    │       └──> Footer      ({ name, year })
    │
    └── 无状态管理库，纯 props 传递

Hook 层:
    useScrollSpy()     → Navigation 使用，返回当前激活的 section
    useTypewriter()    → Hero 使用，返回当前显示的文字
    useCountUp()       → About 使用，返回动画中的数字
```

## 九、响应式策略

| 断点 | 布局调整 |
|------|----------|
| ≥ 1440px | 标准布局，内容区 1200px |
| 1024px - 1439px | 内容区 960px，Skills 3列→2列 |
| ≤ 768px | 单列布局，导航折叠（暂不要求，题目只需桌面端） |

> **虽然题目只要求桌面端，但用 `max-width` + `%` 做弹性布局，自然具备一定响应能力，属于隐性加分。**
