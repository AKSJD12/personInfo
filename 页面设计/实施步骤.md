
## 第四步：编写自定义 Hooks（预计 15 分钟）

### 4.1 `useScrollSpy` — 滚动监听

文件 `src/hooks/useScrollSpy.js`：

```
功能：
1. 接收 section id 列表
2. 用 Intersection Observer 监听各 section
3. 返回当前在视口中的 section id
4. 供 Navigation 使用，高亮当前导航项
```

### 4.2 `useTypewriter` — 打字机效果

文件 `src/hooks/useTypewriter.js`：

```
功能：
1. 接收文本数组和打字速度
2. 逐字输出，循环播放
3. 返回当前显示的文本和闪烁光标状态
4. 供 Hero 使用
```

### 4.3 `useCountUp` — 数字递增

文件 `src/hooks/useCountUp.js`：

```
功能：
1. 接收目标数字和持续时间
2. 用 requestAnimationFrame 实现缓动动画
3. 返回当前动画中的数值
4. 供 About 的数字卡片使用
```

> ✅ 验收：每个 Hook 可在组件中正常调用，无报错

## 第六步：完善动效 & 细节（预计 40 分钟）

### 6.1 滚动渐入动画（全局）

```
- 所有 section 默认 opacity: 0, translateY: 40px
- IntersectionObserver 监听，进入视口 → opacity: 1, translateY: 0
- transition: all 0.6s ease-out
```

### 6.2 粒子背景（Hero）

```
- 用 Canvas 2D 绘制 100 个随机粒子
- 粒子缓慢上升 + 随机横向漂移
- 粒子间距离 < 阈值时连线（形成网络效果）
- 鼠标靠近粒子时排斥
- requestAnimationFrame 循环绘制
```

### 6.3 头像光晕

```
- 头像外层放一个伪元素
- border-radius: 50%
- 渐变边框 + rotate 动画
- @keyframes spin { from { transform: rotate(0deg) } to { transform: rotate(360deg) } }
```

### 6.4 其他细节

```
□ 导航栏滚动时背景加深（没到顶部时半透明→实色）
□ 按钮 hover 时内阴影发光
□ 卡片 hover 时边框渐变
□ 滚动条美化（::-webkit-scrollbar）
□ hover 过渡统一 0.3s ease
```

> ✅ 验收：所有动效流畅（60fps），无卡顿或闪烁

## 第八步：部署上线到 Vercel（预计 20 分钟）

### 8.1 初始化 Git 仓库

```bash
cd personal-site
git init
git add .
git commit -m "feat: 个人介绍网站初始版本"
```

### 8.2 推送到 GitHub

```bash
# 在 GitHub 上创建新仓库（不要勾选 README）
git remote add origin https://github.com/你的用户名/personal-site.git
git branch -M main
git push -u origin main
```

### 8.3 Vercel 部署

```
1. 打开 vercel.com → 用 GitHub 登录
2. 点击 "New Project"
3. 选择 personal-site 仓库
4. 框架自动识别为 Vite
5. 点击 "Deploy"
6. 等待 30 秒 → 获得 https://xxx.vercel.app 链接
```

### 8.4 验证

```
□ 浏览器打开 Vercel 链接，网站正常显示
□ 图片/图标/动效均正常
```
